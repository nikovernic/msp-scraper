# Story 6.3: Crew Authentication & Session Management

## Status
Done

## Story
**As a** crew member with a claimed profile,
**I want** to sign in to my account and maintain an authenticated session,
**so that** I can access my profile edit interface and manage my information.

## Acceptance Criteria
1. Sign-in page displays at `/signin` route
2. Sign-in form includes email and password fields
3. Form validation prevents submission with invalid email or empty password
4. Sign-in API endpoint (POST /api/auth/signin) authenticates user with Supabase Auth
5. API creates authenticated session using Supabase Auth
6. API redirects to profile edit page on successful sign-in
7. API handles errors: invalid credentials (401), account not found (404)
8. Sign-out functionality available (POST /api/auth/signout)
9. Sign-out clears session and redirects to homepage
10. Protected routes check authentication status (middleware or server component)
11. Unauthenticated users are redirected to sign-in page when accessing protected routes
12. Session persistence across page refreshes
13. All code follows coding standards and project structure

```sql
id UUID PRIMARY KEY,

```

### API Specifications
[Source: architecture/api-specification.md]

POST `/api/auth/signin` endpoint:
- Request body:
  ```typescript
  {
    email: string;      // Required, email format
    password: string;   // Required
  }
  ```
- Responses:
  - 200: Sign-in successful, returns redirect URL
  - 400: Invalid request (validation error)
  - 401: Invalid credentials
  - 404: Account not found
  - 500: Server error

POST `/api/auth/signout` endpoint:
- Request body: None
- Responses:
  - 200: Sign-out successful, redirects to homepage
  - 500: Server error

### External APIs
[Source: architecture/external-apis.md#Supabase Auth]

Supabase Auth:
- Purpose: User authentication and session management
- Documentation: https://supabase.com/docs/guides/auth
- Integration: Use Supabase client for auth operations
- Key Operations:
  - `supabase.auth.signInWithPassword()` - Sign in user
  - `supabase.auth.signOut()` - Sign out user
  - `supabase.auth.getSession()` - Get current session
  - `supabase.auth.getUser()` - Get current user
- Integration Notes:
  - Use server-side Supabase client for API routes
  - Sessions are managed by Supabase Auth (cookies)
  - Session persistence is handled automatically by Supabase
