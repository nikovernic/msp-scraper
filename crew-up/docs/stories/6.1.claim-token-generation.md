# Story 6.1: Profile Claim Token Generation & Claim Invitation Email System

## Status
Done

## Story
**As an** admin,
**I want** to generate claim tokens and send claim invitation emails to crew members when creating profiles,
**so that** crew members can claim and manage their own profiles.

## Acceptance Criteria
1. Claim token generation service creates unique, secure tokens (32+ character random string)
2. Claim tokens are stored in profiles table with expiration (30 days from generation)
3. Claim tokens are unique across all profiles (enforced by database constraint)
4. Admin API endpoint (POST /api/admin/profiles/[id]/send-claim-invitation) generates token and sends email
5. Claim invitation email is sent via Resend API using React Email template
6. Email includes claim link with token: `/claim/[token]`
7. Email template includes crew member name and profile information
8. Email service handles errors gracefully (logs errors, doesn't fail profile creation)
9. Token generation can be triggered manually by admin for existing profiles
10. All code follows coding standards and project structure

### Data Models
[Source: architecture/data-models.md#Profile]

Profile interface includes:
- `id`: UUID - Primary key
- `claim_token`: string | null - Unique token for profile claiming
- `claim_token_expires_at`: string | null - Token expiration timestamp
- `is_claimed`: boolean - Whether profile has been claimed
- `contact_email`: string - Email address to send claim invitation to
- `name`: string - Crew member name (for email template)
- `slug`: string - For generating profile link in email

### Database Schema
[Source: architecture/database-schema.md#profiles]

Profiles table already includes:
```sql
claim_token TEXT UNIQUE,
claim_token_expires_at TIMESTAMPTZ,
is_claimed BOOLEAN DEFAULT FALSE,
```

Index available:
- `idx_profiles_claim_token ON public.profiles(claim_token) WHERE claim_token IS NOT NULL`

### API Specifications
[Source: architecture/api-specification.md]

POST `/api/admin/profiles/{id}/send-claim-invitation` endpoint:
- Path parameter: `id` (UUID) - Profile ID
- Authentication: Required (admin role)
- Request body: None (token generated automatically)
- Responses:
  - 200: Claim invitation sent successfully
  - 400: Profile already claimed or invalid request
  - 401: Unauthorized (not admin)
  - 404: Profile not found
  - 500: Server error

### External APIs
[Source: architecture/external-apis.md#Resend Email API]

Resend Email API:
- Purpose: Send transactional emails (claim invitations)
- Documentation: https://resend.com/docs
- Base URL: `https://api.resend.com`
- Authentication: API key in environment variable (`RESEND_API_KEY`)
- Rate Limits: 100 emails/day on free tier, 50,000/month on paid plans
- Key Endpoint: `POST /emails` - Send email with React Email template
- Integration Notes:
  - Use React Email for template design
  - Store API key in Vercel environment variables
  - Implement retry logic for failed sends
  - Log all email sends for debugging
