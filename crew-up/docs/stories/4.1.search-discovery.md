# Story 4.1: Search & Discovery

## Status
Done

## Story
**As a** producer,
**I want** to search for crew members using free-text queries and filters (location, role, # of years in the industry),
**so that** I can quickly find qualified crew members for my production needs.

## Acceptance Criteria
1. Producers can search for crew using free-text queries (e.g., "AC in Nashville", "gaffer Atlanta")
2. Search query parsing extracts role and location information from natural language queries
3. Full-text search implementation uses PostgreSQL GIN index to search across name, role, bio, and location fields
4. Search filters are available for: location (city, state), role, and # of years in the industry (numeric range)
5. Search results page displays profiles in a card-based layout showing name, photo, primary role, location, and top 3 credits
6. Search results are paginated with 20 results per page
7. Profile cards include "View Profile" link to individual profile pages
8. Search results page is server-side rendered (SSR) for SEO optimization
9. Search functionality is accessible from homepage search interface
10. All code follows coding standards and project structure

### Data Models
[Source: architecture/data-models.md#Profile]

Profile interface includes:
- `id`: UUID
- `name`: string
- `primary_role`: string
- `primary_location_city`: string
- `primary_location_state`: string (2-letter code)
- `bio`: string | null (max 250 chars)
- `photo_url`: string | null
- `union_status`: 'union' | 'non-union' | 'either' | null
- `years_experience`: number | null
- `secondary_roles`: string[] | null
- `additional_markets`: Array<{city: string; state: string}> | null
- `slug`: string
- `credits`: Credit[] (relationship, not in base Profile type)

Credit interface includes:
- `id`: UUID
- `project_title`: string
- `role`: string
- `project_type`: enum
- `year`: number
- `display_order`: number (for ordering)

### Database Schema
[Source: architecture/database-schema.md]

Full-text search index already exists:
```sql
CREATE INDEX idx_profiles_search ON public.profiles 
    USING gin(to_tsvector('english', 
        coalesce(name, '') || ' ' || 
        coalesce(primary_role, '') || ' ' || 
        coalesce(bio, '') || ' ' ||
        coalesce(primary_location_city, '') || ' ' ||
        coalesce(primary_location_state, '')
    ));
```

Additional indexes available:
- `idx_profiles_location` on (primary_location_city, primary_location_state)
- `idx_profiles_role` on (primary_role)
- `idx_credits_profile` on (profile_id)
- `idx_credits_display_order` on (profile_id, display_order)

For full-text search, use PostgreSQL `to_tsquery()` with the GIN index. Supabase PostgREST supports `.textSearch()` method or RPC functions.

### API Specifications
[Source: architecture/api-specification.md#/profiles]

GET `/api/profiles` endpoint:
- Query parameters:
  - `q`: string (optional) - Search query text
  - `role`: string (optional) - Filter by role
  - `city`: string (optional) - Filter by city
  - `state`: string (optional) - Filter by state (2-letter code)
  - `years_experience_min`: integer (optional) - Minimum years of experience
  - `years_experience_max`: integer (optional) - Maximum years of experience
  - `page`: integer (default: 1, min: 1)
  - `limit`: integer (default: 20, min: 1, max: 100)
- Response:
  ```typescript
  {
    profiles: Profile[],
    pagination: {
      page: number,
      limit: number,
      total: number,
      totalPages: number
    }
  }
  ```
