# Story 2.1: Core Profile System

## Status
Done

## Story
**As a** admin,
**I want** to create and manage crew profiles with photos, contact info, websites, credits, and SEO-friendly URLs,
**so that** crew members can be discovered and contacted by producers.

## Acceptance Criteria
1. Admin can create new profiles via API endpoint with all required fields (name, primary role, location, contact email, phone number)
2. Profile creation generates unique, SEO-friendly slug (format: `{name}-{role}-{city}`)
3. Admin can upload profile photos to Supabase Storage
4. Admin can add, edit, and delete credits for profiles
5. Basic profile display page exists at `/crew/[slug]` with server-side rendering
6. Profile page displays all profile information including photo, bio, credits, and contact information
7. All profile operations use proper authentication and authorization (admin-only for creation/editing)
8. All code follows coding standards and project structure

### API Specifications
[Source: docs/architecture/api-specification.md]

**POST /api/admin/profiles:**
- Admin-only endpoint
- Request body: `CreateProfileRequest` schema
- Response: 201 with created Profile object
- Errors: 400 (validation), 401 (unauthorized), 403 (forbidden)

**GET /api/profiles/{slug}:**
- Public endpoint (will be created in this story for profile page)
- Note: This endpoint is not explicitly in api-specification.md but is needed for profile display page (AC5, AC6)
- Response: 200 with ProfileWithCredits object
- Errors: 404 (not found)

**POST /api/admin/profiles/{id}/credits:**
- Admin-only endpoint
- Request body: Credit data
- Response: 201 with created Credit object

**PUT /api/admin/profiles/{id}/credits/{creditId}:**
- Admin-only endpoint
- Request body: Updated Credit data
- Response: 200 with updated Credit object

**DELETE /api/admin/profiles/{id}/credits/{creditId}:**
- Admin-only endpoint
- Response: 204 No Content

- API routes in `apps/web/app/api/`

- Components in `apps/web/components/`
- All code must follow unified project structure

**Slug Generation Rules:**
- Format: `{name}-{role}-{city}` (all lowercase, spaces to hyphens)
- Remove special characters
- Handle duplicates by appending `-2`, `-3`, etc.
- Max length: 100 characters (truncate if needed)

**Photo Upload Constraints:**
- Allowed types: jpg, jpeg, png, webp
- Max file size: 5MB
- Store in Supabase Storage bucket created in Story 1.1
- Generate unique filename (UUID + extension)
