## Should-Fix Issues (Important Quality Improvements)

1. **Experience Level Mapping Clarification** (Task 3, Task 7, AC4)
   - **Issue:** The story mentions "experience level (entry, mid, senior)" in AC4 and Task 7, but Task 3 uses `years_experience_min` and `years_experience_max` in the API schema. The mapping between Entry/Mid/Senior and year ranges is not defined.
   - **Impact:** Developer must make a reasonable implementation decision (e.g., Entry: 0-3, Mid: 4-10, Senior: 10+)
   - **Recommendation:** Add a note in Dev Notes or Task 7 clarifying the experience level to years mapping, OR specify that the UI can map Entry/Mid/Senior to min/max years ranges (e.g., Entry: 0-3, Mid: 4-10, Senior: 10+)
   - **Priority:** Medium (developer can make reasonable decision, but clarification would improve consistency)

2. **Search Rate Limiting Consideration** (Task 3)
   - **Issue:** Search API endpoint doesn't mention rate limiting considerations
   - **Impact:** Public search endpoint could potentially be abused
   - **Recommendation:** Consider adding a note about rate limiting for search endpoint (optional, performance consideration)
   - **Priority:** Low (nice-to-have, not blocking for MVP)
