## Story
**As a** crew member with a claimed profile,
**I want** to edit my profile information through a web interface,
**so that** I can keep my information current and accurate.

## Acceptance Criteria
1. Profile edit page displays at `/profile/edit` route (protected, requires authentication)
2. Page loads current profile data for authenticated crew member
3. Edit form includes all editable profile fields (name, role, location, bio, contact info, social links, union status, experience, secondary roles, additional markets)
4. Form validation prevents submission with invalid data (email format, URL format, character limits)
5. Photo upload functionality allows crew to update profile photo
6. Profile update API endpoint (PUT /api/profiles/me) accepts and validates profile updates
7. API updates profile in database (only for authenticated user's own profile)
8. API returns success response with updated profile data
9. API handles errors: unauthorized (401), profile not found (404), validation errors (400)
10. Form shows success message on successful update
11. Form shows error message on validation or API errors
12. All code follows coding standards and project structure

### Data Models
[Source: architecture/data-models.md#Profile, Credit]

Profile interface includes all editable fields:
- `name`: string
- `primary_role`: string
- `primary_location_city`: string
- `primary_location_state`: string
- `bio`: string | null (max 250 chars)
- `photo_url`: string | null
- `contact_email`: string
- `contact_phone`: string | null
- `portfolio_url`: string | null
- `website`: string | null
- `instagram_url`: string | null
- `vimeo_url`: string | null
- `union_status`: 'union' | 'non-union' | 'either' | null
- `years_experience`: number | null
- `secondary_roles`: string[] | null
- `additional_markets`: Array<{city: string; state: string}> | null

Credit interface (for credits management):
- `id`: UUID
- `profile_id`: UUID
- `project_title`: string
- `role`: string
- `project_type`: enum
- `year`: number
- `production_company`: string | null
- `director`: string | null
- `display_order`: number

### Database Schema
[Source: architecture/database-schema.md#profiles, credits]

Profiles table includes all editable fields with appropriate constraints:
- `bio TEXT CHECK (char_length(bio) <= 250)`
- `union_status TEXT CHECK (union_status IN ('union', 'non-union', 'either'))`
- RLS policy: "Users can update own profile" - `UPDATE USING (auth.uid() = user_id)`

Credits table:
- RLS policy: "Users can manage own profile credits" - `FOR ALL USING (EXISTS (SELECT 1 FROM public.profiles WHERE id = profile_id AND user_id = auth.uid()))`

### API Specifications
[Source: architecture/api-specification.md]

PUT `/api/profiles/me` endpoint:
- Authentication: Required (authenticated crew user)
- Request body:
  ```typescript
  {
    name?: string;
    primary_role?: string;
    primary_location_city?: string;
    primary_location_state?: string;
    bio?: string | null;  // Max 250 chars
    contact_email?: string;
    contact_phone?: string | null;
    portfolio_url?: string | null;
    website?: string | null;
    instagram_url?: string | null;
    vimeo_url?: string | null;
    union_status?: 'union' | 'non-union' | 'either' | null;
    years_experience?: number | null;
    secondary_roles?: string[] | null;
    additional_markets?: Array<{city: string; state: string}> | null;
    photo_url?: string | null;
  }
  ```
- Responses:
  - 200: Profile updated successfully, returns updated profile
  - 400: Invalid request (validation error)
  - 401: Unauthorized (not authenticated or not own profile)
  - 404: Profile not found
  - 500: Server error
