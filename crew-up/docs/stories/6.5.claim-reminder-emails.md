# Story 6.5: Claim Reminder Email System

## Status
Ready for Review

## Story
**As an** admin,
**I want** the system to automatically send reminder emails to crew members who haven't claimed their profiles,
**so that** more profiles get claimed and crew members can manage their own information.

## Acceptance Criteria
1. Scheduled job or API endpoint identifies unclaimed profiles with valid claim tokens
2. System sends reminder email 7 days after claim invitation was sent (if not claimed)
3. System sends reminder email 14 days after claim invitation was sent (if not claimed)
4. Reminder emails use React Email template with claim link
5. Reminder emails include profile information and clear call-to-action
6. System tracks reminder emails sent to avoid duplicate reminders
7. System does not send reminders for expired tokens (30+ days old)
8. System does not send reminders for already-claimed profiles
9. Reminder email service handles errors gracefully (logs errors, continues processing)
10. Admin can manually trigger reminder emails via API endpoint (POST /api/admin/profiles/[id]/send-claim-reminder)
11. All code follows coding standards and project structure

### Data Models
[Source: architecture/data-models.md#Profile]

Profile interface includes:
- `id`: UUID - Primary key
- `is_claimed`: boolean - Whether profile has been claimed
- `claim_token`: string | null - Unique token for profile claiming
- `claim_token_expires_at`: string | null - Token expiration timestamp (30 days from generation)
- `contact_email`: string - Email address to send reminder to
- `name`: string - Crew member name (for email template)
- `slug`: string - For generating profile link in email

Reminder tracking (new):
- `reminder_sent_at_7days`: timestamp | null - When 7-day reminder was sent
- `reminder_sent_at_14days`: timestamp | null - When 14-day reminder was sent

### API Specifications
[Source: architecture/api-specification.md]

POST `/api/admin/reminders/process` endpoint:
- Authentication: Required (admin role)
- Request body: None
- Responses:
  - 200: Reminders processed successfully, returns count of reminders sent
  - 401: Unauthorized (not admin)
  - 500: Server error

POST `/api/admin/profiles/{id}/send-claim-reminder` endpoint:
- Path parameter: `id` (UUID) - Profile ID
- Authentication: Required (admin role)
- Request body: None (reminder type determined automatically)
- Responses:
  - 200: Reminder sent successfully
  - 400: Profile already claimed or invalid request
  - 401: Unauthorized (not admin)
  - 404: Profile not found
  - 500: Server error
